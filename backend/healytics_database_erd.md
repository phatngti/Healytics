## Table-by-Table Reference

### Domain 1: Authentication & User Management

---

#### `account`
The **central identity table**. Every user, employee, partner, and admin has an account row. Drives JWT authentication and RBAC.

| Column | Type | Purpose |
|--------|------|---------|
| `id` | UUID (PK) | Auto-generated primary identifier |
| `email` | VARCHAR(255), UNIQUE | Login credential, indexed for lookups |
| `username` | VARCHAR, nullable | Optional display/login name (added later) |
| `password_hash` | VARCHAR(255), nullable | Bcrypt-hashed password (null for OAuth users) |
| `refresh_token_hash` | TEXT, nullable | Hashed JWT refresh token for session persistence |
| `role` | ENUM | `user`, `employee`, `health_partner`, `admin` — determines API access tier |
| `survey` | JSONB, nullable | Stores onboarding survey responses |
| `is_active` | BOOLEAN | Account enabled/disabled flag |
| `created_at / updated_at` | TIMESTAMPTZ | Audit timestamps |
| `deleted_at` | TIMESTAMPTZ, nullable | Soft delete marker |

---

#### `address`
Stores physical address information, linked 1:1 to user profiles.

| Column | Type | Purpose |
|--------|------|---------|
| `street` | VARCHAR(255) | Street address |
| `ward` | VARCHAR(255) | Administrative ward |
| `district` | VARCHAR(255) | District/county |
